# 现状
目前我对typescript有一定的基础，这里做复习和查漏补缺

# 核心基础
- any&unknown
尽量使用unknown，不要使用any，any会导致类型检查的严格性变弱

- 类型收窄
in typeof instanceof 自定义类型守卫

# 对象类型
-  interface & type
| 能力   | interface | type |
| ---- | --------- | ---- |
| 对象定义 | √         | √    |
| 联合类型 | ×         | √    |
| 交叉类型 | ×         | √    |
| 声明合并 | √         | ×    |
- 索引签名
interface Dict {
  [key: string]: string;
}
- Record
内置的工具类型
Record<string, User>
- Partial&Required
创造一个类型，让它的所有属性变为可选的或者必选的
Partial<User>
Required<User>

# 泛型
一种允许我们在定义`函数`、`接口`或`类`时不预先指定具体类型，而在使用时再传入类型的机制。

# 高级类型
- keyof & typeof
- 索引访问类型
type Name =
  User["name"];

# Utility Types
工具类型，内置的封装库，用于快速进行常见的类型转换
- Pick & Omit
挑选&剔除
- Extract & Exclude
提取&排除 联合类型

# 工程化TS
- tsconfig.json
{
  "strict": true
}
- Path Alias
{
  "paths": {
    "@/*": ["src/*"]
  }
}
- module
{
  "module": "ESNext"
}
代表支持‌最新且稳定‌的 ECMAScript 模块特性。
