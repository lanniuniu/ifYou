- table
‌主键（Primary Key）‌：唯一标识表中每一行数据的列，不能为空且不能重复（通常是 id）。
‌外键（Foreign Key）‌：用于建立表与表之间关联的列，指向另一张表的主键。
‌索引（Index）‌：类似于书籍的目录，用于加速数据查询速度。
```
// 创建表
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY AUTO_INCREMENT, // 自增主键
    username VARCHAR(255) NOT NULL, // 非空
    email VARCHAR(255) UNIQUE, // 唯一 可以为空
    age INT
)

// 插入数据
INSERT INTO users (username, email, age) 
VALUES ('alice', 'alice@example.com', 25);

// 查询
SELECT * FROM users WHERE age > 20;

-- 添加新列
ALTER TABLE users ADD COLUMN phone VARCHAR(20);

-- 修改列类型
ALTER TABLE users MODIFY COLUMN age SMALLINT;

// 删除表
DROP TABLE users;

// 注释
COMMENT ""
```

- 索引
分类：数据结构&物理存储&逻辑约束
按逻辑约束分类：主键索引、唯一索引、普通索引、组合索引、其他高级索引
| 索引类型 | 适用场景 | 注意事项 |
| --- | --- | --- |
| ‌主键索引‌ | 几乎每张表必备 | 尽量选择短小、自增的整数类型 |
| ‌唯一索引‌ | 业务上不能重复的字段（如手机号、订单号） | 区分于主键，允许为空 |
| ‌普通/组合索引‌ | 高频查询的 WHERE、JOIN、ORDER BY 字段 | 遵循最左前缀原则，避免过多索引影响写性能 |
| ‌全文索引‌ | 大文本搜索 | 替代 LIKE %...% |
‌不要过度索引‌：每个索引都会降低写操作的速度，并占用磁盘空间。
‌区分度高优先‌：选择区分度高的列（如身份证、邮箱）建立索引，性别（只有男/女）建立索引意义不大。
‌覆盖索引优化‌：如果查询的列都在索引中，无需回表，性能极佳。

- 约束(Constraint)
分类：实体完整性约束、参照完整性约束、域完整性约束
```
CREATE TABLE employees (
    id INT AUTO_INCREMENT PRIMARY KEY,          -- 主键约束
    email VARCHAR(100) NOT NULL UNIQUE,         -- 非空 + 唯一约束
    department_id INT,
    salary DECIMAL(10, 2) DEFAULT 0.00,         -- 默认值约束
    age INT CHECK (age >= 18 AND age <= 65),    -- 检查约束
    CONSTRAINT fk_dept                          -- 外键约束（命名规范）
        FOREIGN KEY (department_id) 
        REFERENCES departments(id)
);
```

- 数字

DECIMAL：以字符串形式存储，‌精度绝对准确‌，不会丢失精度。 金额
FLOAT：占用空间小，计算速度快，但存在‌精度丢失‌问题。
DOUBLE
- 字符串
CHAR 定长 存取速度极快 如果存入短数据，剩余空间会用空格填充，造成浪费
VARCHAR 变长 节省存储空间，适合长度变化较大的数据 于长度不固定，存取速度略慢于 CHAR，且频繁修改可能导致页分裂。
TEXT 长文本 存储较长的文本数据，最大长度为 ‌65,535 字节 不能有默认值，检索效率低于 VARCHAR
LONGTEXT 用于存储极大的文本数据，最大长度为 ‌4GB 长篇博客文章、JSON 文档、Base64 编码的大文件。
