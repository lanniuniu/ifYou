- table
‌主键（Primary Key）‌：唯一标识表中每一行数据的列，不能为空且不能重复（通常是 id）。
‌外键（Foreign Key）‌：用于建立表与表之间关联的列，指向另一张表的主键。
‌索引（Index）‌：类似于书籍的目录，用于加速数据查询速度。
```
// 创建表
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY AUTO_INCREMENT, // 自增主键
    username VARCHAR(255) NOT NULL, // 非空
    email VARCHAR(255) UNIQUE, // 唯一 可以为空
    age INT
)

// 插入数据
INSERT INTO users (username, email, age) 
VALUES ('alice', 'alice@example.com', 25);

// 查询
SELECT * FROM users WHERE age > 20;

-- 添加新列
ALTER TABLE users ADD COLUMN phone VARCHAR(20);

-- 修改列类型
ALTER TABLE users MODIFY COLUMN age SMALLINT;

// 删除表
DROP TABLE users;

// 注释
COMMENT ""
```

- 索引
分类：数据结构&物理存储&逻辑约束
按逻辑约束分类：主键索引、唯一索引、普通索引、组合索引、其他高级索引
| 索引类型 | 适用场景 | 注意事项 |
| --- | --- | --- |
| ‌主键索引‌ | 几乎每张表必备 | 尽量选择短小、自增的整数类型 |
| ‌唯一索引‌ | 业务上不能重复的字段（如手机号、订单号） | 区分于主键，允许为空 |
| ‌普通/组合索引‌ | 高频查询的 WHERE、JOIN、ORDER BY 字段 | 遵循最左前缀原则，避免过多索引影响写性能 |
| ‌全文索引‌ | 大文本搜索 | 替代 LIKE %...% |
‌不要过度索引‌：每个索引都会降低写操作的速度，并占用磁盘空间。
‌区分度高优先‌：选择区分度高的列（如身份证、邮箱）建立索引，性别（只有男/女）建立索引意义不大。
‌覆盖索引优化‌：如果查询的列都在索引中，无需回表，性能极佳。

```
// 组合/复合索引
ALTER TABLE users ADD INDEX idx_name_age (username, age);
```

- 约束(Constraint)
分类：实体完整性约束、参照完整性约束、域完整性约束
```
CREATE TABLE employees (
    id INT AUTO_INCREMENT PRIMARY KEY,          -- 主键约束
    email VARCHAR(100) NOT NULL UNIQUE,         -- 非空 + 唯一约束
    department_id INT,
    salary DECIMAL(10, 2) DEFAULT 0.00,         -- 默认值约束
    age INT CHECK (age >= 18 AND age <= 65),    -- 检查约束
    CONSTRAINT fk_dept                          -- 外键约束（命名规范）
        FOREIGN KEY (department_id) 
        REFERENCES departments(id)
);
```

# SQL 查询
- WHERE
- LIKE
LIKE 是 SQL 中用于‌模糊匹配字符串‌的运算符，通常配合 WHERE 子句使用，通过通配符来搜索符合特定模式的数据。
%: 匹配任意数量的字符（包括零个字符）
_: 匹配单个字符
索引失效‌：当 LIKE 以 % 开头时（如 '%keyword'），MySQL 无法使用索引，MySQL InnoDB 引擎的普通索引，
    B+Tree 索引的最左前缀匹配原则‌。会导致全表扫描，性能极差。
优化建议‌：如果业务需要频繁做后缀模糊搜索，可考虑使用全文索引（FULLTEXT）或 Elasticsearch 等搜索引擎。
如果待搜索的字符串本身包含 % 或 _，需要使用 ESCAPE 关键字指定转义符
-- 查找包含“50%”字样的记录
SELECT * FROM products WHERE discount_info LIKE '%50\%%' ESCAPE '\';

- DISTINCT
DISTINCT 是 SQL 中用于‌去除查询结果中重复行‌的关键字。
DISTINCT 必须紧跟 SELECT 之后、字段名之前使用
在 DISTINCT 中，多个 NULL 值被视为‌相同‌，最终只保留一个 NULL。
‌先过滤再去重‌：大数据量场景下，先用 WHERE 缩小数据范围，再执行 DISTINCT，避免全表扫描。

- GROUP BY
GROUP BY 是 SQL 中用于‌将数据按指定列分组，并结合聚合函数进行统计计算‌的核心子句。
SQL 的执行顺序是：WHERE → GROUP BY → HAVING → ORDER BY。
```
SELECT Customer, SUM(OrderPrice) // 返回客户名和对应的总额
FROM Orders // 表名
GROUP BY Customer; // 相同客户合并为1组
```
HAVING和WHERE区别：
WHERE是过滤数据，HAVING是过滤分组后的聚合函数结果

- CASE WHEN
类似if else
```
-- 根据分数评定等级
SELECT
    name,
    score,
    CASE
        WHEN score >= 90 THEN '优秀'
        WHEN score >= 80 THEN '良好'
        WHEN score >= 60 THEN '及格'
        ELSE '不及格'
    END AS grade
FROM students;
```
用途：‌动态排序、行列转换‌、数据分类
必须 END 结尾

- UNION
UNION 可以将两个或多个 SELECT 查询的结果纵向拼接成一个结果集，但要求各查询的‌列数相同‌且对应列的‌数据类型兼容‌。
```
SELECT column1, column2 FROM table1
UNION
SELECT column1, column2 FROM table2;
```
UNION 与 UNION ALL 的关键区别
UNION： 会去重，由于需要扫描比较每一行来判断是否重复，执行速度相对较慢。
UNION ALL 不会去重。省去了去重和排序的额外开销，处理大数据时性能优势明显。
